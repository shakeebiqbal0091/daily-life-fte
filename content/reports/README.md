# Reports Directory

This directory stores all structured reports and summaries generated by the Daily Life Productivity Digital FTE system.

## Purpose

Generated reports including:
- Weekly review reports
- Research summaries
- Planning documents
- Performance analyses
- Progress tracking reports

## Report Types

### Weekly Review Reports
Comprehensive weekly analyses including:
- Accomplishments and achievements
- Goal progress and completion
- Lessons learned and insights
- Next week planning and priorities

### Research Reports
Detailed research summaries including:
- Industry analysis and trends
- Learning resource evaluations
- Market research findings
- Competitive analysis

### Planning Documents
Strategic planning documents including:
- Weekly and monthly planning
- Goal setting and tracking
- Project planning and timelines
- Resource allocation

### Performance Reports
Performance tracking including:
- Productivity metrics
- Goal achievement rates
- Time management analysis
- Skill development progress

## File Naming Convention

```
YYYY-MM-DD_report-type.md

Examples:
2026-02-12_weekly_review.md
2026-02-12_research_summary.md
2026-02-12_planning_document.md
2026-02-12_performance_analysis.md
```

## Metadata Standard

Each report content file should include:

```markdown
---
title: "Report Title"
created: "2026-02-12"
generated_by: "Weekly Reviewer Skill / Learning Assistant Agent"
report_type: "weekly_review/research_summary/planning_document/performance_analysis"
status: "draft/final"
related_workflow: "weekly-review/planning"
audience: "personal/team/management"
---

# Report Body
```

## Report Structure

### Standard Report Format
```markdown
# Report Title
**Date**: 2026-02-12
**Generated By**: Weekly Reviewer Skill
**Audience**: Personal

## Executive Summary
Brief overview of key findings and recommendations.

## Detailed Analysis
In-depth analysis of report subject matter.

## Key Findings
Summary of main discoveries and insights.

## Recommendations
Actionable recommendations based on findings.

## Next Steps
Specific actions to be taken.
```

## Report Generation Workflow

1. **Data Collection**: Gather relevant data and information
2. **Analysis**: Analyze data and identify patterns
3. **Drafting**: Create report structure and content
4. **Review**: User reviews and approves report
5. **Distribution**: Report shared with relevant parties
6. **Action**: Implement recommendations and next steps
